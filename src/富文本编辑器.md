# Astro + React + Tiptap MDX 富文本编辑器完整开发方案

## 1. 项目概述

开发一款可嵌入网页的产品级编辑器，面向普通用户时表现为类似 Notion、飞书文档或 Word 的富文本编辑器，底层统一使用 MDX 存储。

主要能力：

- 可嵌入 Astro 页面
- 所见即所得富文本编辑
- 导入、编辑、校验和导出 MDX
- 强大的图片上传与编辑
- 可视化数学公式编辑
- 完整的表格操作
- 自定义 MDX 业务组件
- MDX 源码和最终预览模式
- 自动保存、版本历史和草稿发布
- 可选实时协作、评论和 AI 写作
- 尽量使用开源、自托管方案

### 2.2 为什么使用 Astro + React

- Astro 负责内容站点、路由、SSR/SSG和低 JavaScript 输出。
- React 只加载在编辑器及必要的交互组件中。
- Tiptap 对 React 有正式支持。
- MDX 和 JSX/React 组件生态结合更自然。
- 编辑器、公式、属性面板及第三方 UI 示例更多。
- 公开页面不需要加载完整 React 和 Tiptap。

## 4. 内容模型

### 4.1 唯一事实来源

数据库中的原始 MDX 字符串是永久内容格式。

```text
MDX：永久存储、导入、导出和版本管理
Tiptap JSON：运行时编辑状态和可选缓存
Yjs：启用协作时的实时状态
HTML：展示层派生结果
```

不能把 MDX、Tiptap JSON 和 HTML 同时作为可独立修改的主数据。

### 4.2 受控 MDX

支持：

- CommonMark
- GitHub Flavored Markdown
- Frontmatter
- 行内和块级公式
- 白名单 JSX 组件
- JSON 可序列化属性
- 图片、表格、链接、代码块
- 指令式业务节点

示例：

```mdx
---
title: 二次方程
description: 二次方程求根公式
---

# 二次方程

设有方程 $ax^2+bx+c=0$。

$$
x=\frac{-b\pm\sqrt{b^2-4ac}}{2a}
$$

<Callout type="warning">当判别式小于零时，方程没有实数解。</Callout>

<Figure assetId="asset-100" alt="函数图像" caption="图 1：二次函数图像" />
```

默认禁止：

- 动态 `import`
- 任意 `export`
- 任意 JavaScript 语句
- 任意函数调用
- 不受限制的 `{expression}`
- 未注册组件的执行
- 组件内部副作用

## 6. Astro 集成

安装：

```bash
pnpm astro add react
pnpm astro add mdx
```

编辑页面：

```astro
---
import AdminLayout from '../../../layouts/AdminLayout.astro';
import DocumentEditor from '../../../components/editor/DocumentEditor';
import { getDocument } from '../../../lib/documents';

const { id } = Astro.params;
const document = await getDocument(id);
---

<AdminLayout title={`编辑：${document.title}`}>
  <DocumentEditor
    client:only="react"
    documentId={document.id}
    initialMdx={document.contentMdx}
    initialVersion={document.version}
  >
    <div slot="fallback">正在加载编辑器……</div>
  </DocumentEditor>
```

## 16. 自动保存

策略：

1. 用户编辑后防抖 1000ms（可配置参数 `autosaveDebounceMs`）。
2. 客户端生成合法 MDX。
3. 调用自动保存接口。
4. 服务端校验 MDX。
5. 使用版本号进行乐观锁检查。
6. 更新草稿。
7. 定期生成完整版本快照（建议间隔 5 分钟）。

### 16.1 保存状态机

```text
              ┌─────→ 已保存 ─────┐
              │                    │
  空闲 ──→ 存在未保存更改 ──→ 正在保存
              │         ↑          │
              │         │          ↓
              │         └──── 保存失败
              │                    │
              └──── 版本冲突 ←─────┘
```

- **已保存**：服务端确认保存成功
- **正在保存**：请求已发出，等待响应
- **存在未保存更改**：有编辑但仍未达到防抖时间
- **保存失败**：网络或服务端错误，可重试
- **版本冲突**：`baseVersion` 与服务端不一致，需用户选择策略

### 16.2 离线与竞态处理

- 使用 `beforeunload` 事件：存在未保存更改时阻止页面关闭，展示浏览器原生确认对话框。
- `beforeunload` 和自动保存的竞态：在 `beforeunload` 中同步调用 `navigator.sendBeacon()` 发送最后一次草稿内容，确保即使页面关闭也能保存。
- 网络断开时：将草稿写入 `localStorage` 作为本地备份，网络恢复后自动同步。避免用户在无感知情况下丢失离线期间的全部编辑内容。
- `localStorage` 中的草稿在服务端确认保存成功后清除。

## 28. 最终结论

正式方案采用：

```text
Astro
+ React Island
+ Tiptap 3
+ 受控 MDX
+ unified/remark/MDAST
+ MathLive
+ KaTeX
+ CodeMirror 6
+ Uppy
+ PostgreSQL
+ S3/MinIO
```

设计原则：

1. MDX 是唯一永久内容格式。
2. Tiptap JSON 只是编辑状态。
3. 不执行用户提供的任意 MDX 代码。
4. 使用白名单组件表达复杂内容。
5. 简单表格使用 GFM，复杂表格使用 MDX 组件。
6. 未知节点必须保留原始源码。
7. 编辑器、源码和预览共享同一套组件 Schema。
8. 协作、评论和文档转换后置实施。
9. 公开页面保持 Astro 的静态输出优势。
10. 将 MDX 往返测试作为项目核心质量门槛。
