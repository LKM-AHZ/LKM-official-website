# 常见的网站登录方式

## 一级：基础登录（最常用）

### 1. 账号密码登录

- 用户名 + 密码
- 邮箱 + 密码
- 手机号 + 密码

### 2. 手机验证码登录

- 手机号/邮箱 + 短信验证码

### 3.平台登陆

- Github登陆

## 二级：进阶安全登录

### 4. 无密码登录

- 邮箱魔法链接（Magic Link）
- 通行密钥（Passkey / WebAuthn）

### 5. 双因素认证（2FA / MFA）

- TOTP 动态验证码（Google Authenticator 等）
- 备用恢复码

# 账户分级设计

## 账户等级定义

| 等级     | 名称       | 说明                   |
| -------- | ---------- | ---------------------- |
| `local`  | 本地账户   | 最低权限，仅本系统可用 |
| `normal` | 普通账户   | 标准用户，2FA 可选     |
| `admin`  | 管理员账户 | 最高权限，2FA 强制     |

---

### local — 本地账户

- 最初仅支持：**用户名 + 密码**
- **可升级**：绑定任意一种非密码登录方式后自动升级为 normal
- 未升级前不可绑定邮箱、手机号、Github OAuth、Passkey
- 不支持 Magic Link
- **不支持 2FA**（无需 TOTP，无恢复码）
- 密码找回：**无**（无邮箱/手机，忘记密码即不可恢复）
- 适用场景：内部工具、本地开发、单机应用

```
注册：用户名 + 密码 → 完成（local）
登录：用户名 + 密码 → 直接进入应用
找回：不支持（忘密即废）

升级规则：
  local 账户在设置页操作以下任一行为时，自动升级为 normal：
  ├── 绑定邮箱（验证通过后）
  ├── 绑定手机号（验证通过后）
  ├── 绑定 Github OAuth（授权成功后）
  └── 绑定 Passkey 通行密钥（创建成功后）
  → 升级后刷新页面即可使用 normal 全部功能（2FA 可选等）
```

---

### normal — 普通账户

- 支持全部登录方式：
  - 用户名/邮箱/手机号 + 密码
  - 手机号/邮箱 + 验证码
  - Github OAuth
  - Magic Link
  - Passkey
- **2FA 可选**：用户自主决定是否开启
- 可绑定邮箱、手机号（用于密码找回）
- 注册时至少绑定邮箱或手机号二者之一

```
注册：
  邮箱/手机号 + 密码 → 验证邮箱/手机号 → 完成
                     → 引导设置 2FA（可选，可跳过）
                     → 引导绑定 Passkey（可选，可跳过）

登录：
  主校验通过 → 检查 2FA 状态：
    ├── 未开启 2FA → 直接进入应用
    └── 已开启 2FA → 跳转 2FA 验证页

找回：
  通过绑定的邮箱/手机号 → 验证码/Magic Link → 重置密码
```

---

### admin — 管理员账户

- 支持全部登录方式（同 normal）
- **2FA 强制**：首次登录或在设置中提升为 admin 时，必须完成 2FA 绑定
- 备用恢复码必须下载/保存后才算 2FA 设置完成
- 未绑定 2FA 的管理员不可登录
- 额外限制：
  - 验证码请求频率限制比 normal 更宽松（允许更多次）
  - 密码复杂度要求更高（长度、字符类型）
  - 敏感操作（删除用户、修改权限）需再次验证 2FA

```
注册/提升为 admin：
  → 主校验通过
  → 检查 2FA 状态：
    ├── 已开启 2FA → 进入应用
    └── 未开启 2FA → 强制跳转 2FA 绑定页
                     ├── 扫码/手动输入密钥 → 绑定 TOTP
                     ├── 下载/复制备用恢复码 → 用户确认已保存
                     └── 完成后 → 进入应用
      （在绑定完成前无法进入应用）

登录：
  主校验通过 → 必须走 2FA → 输入 TOTP 6 位码 → 进入应用
  （admin 不可跳过 2FA，不可勾选「信任设备 30 天」）
```

## 账户等级对比

| 维度             | local                 | normal          | admin              |
| ---------------- | --------------------- | --------------- | ------------------ |
| 登录方式         | 仅用户名+密码         | 全部            | 全部               |
| 密码找回         | 不支持                | 支持            | 支持               |
| 绑定邮箱/手机    | 绑定后自动升级 normal | 至少其一        | 必须               |
| 2FA              | 不支持                | 可选            | 强制               |
| 备用恢复码       | 无                    | 仅开启2FA时下发 | 强制下发并确认保存 |
| 信任设备免2FA    | 无                    | 可勾选          | 不可勾选           |
| 密码复杂度       | 普通                  | 普通            | 高                 |
| 敏感操作二次验证 | 无                    | 可选            | 必须               |

## 账户等级变更

```
local → normal（自动升级）：
  local 用户在设置页绑定任一非密码登录方式时，无需审批，立即自动升级：
  ├── 绑定邮箱（验证通过）  → 自动升级为 normal
  ├── 绑定手机号（验证通过）→ 自动升级为 normal
  ├── 绑定 Github OAuth     → 自动升级为 normal
  └── 绑定 Passkey          → 自动升级为 normal
  → 升级后即获得 normal 全部权限（2FA 可选、密码可找回等）
  → 升级不可逆（local 不能手动降回，除非由 admin 操作）

normal → local（降级）：
  └── 由 admin 手动操作 → 移除所有非密码登录方式 → 降为 local

normal → admin：
  ├── 已开启 2FA → 由 admin 直接提升
  └── 未开启 2FA → 强制绑定 2FA → 才能提升

admin → normal：
  └── 关闭 2FA → 自动降级为 normal（admin 失去 2FA 即失去 admin 资格）

admin → local → normal → admin 变更记录审计日志
```

# 注册/登录/找回流程设计

## 一、注册流程

### 1.1 本地账户注册

```
注册页 → 选择「本地账户」
       → 输入 用户名 + 密码 + 确认密码
       → 注册成功，自动登录
       → 无后续引导（无 2FA、无 Passkey、不可绑定邮箱/手机）
```

### 1.2 普通账户注册（账号密码）

```
注册页 → 选择「普通账户」
       → 输入 用户名 + 邮箱/手机号 + 密码 + 确认密码
       → 邮箱绑定：发送邮箱验证码  |  手机号绑定：发送短信验证码
       → 输入验证码校验
       → 校验通过，注册成功，自动登录
       → 引导设置 2FA（可跳过）、绑定 Passkey（可跳过）
```

### 1.3 普通账户注册（验证码无密码）

```
注册页 → 输入手机号/邮箱
       → 发送验证码（60s 倒计时，限频：1条/分钟，5条/小时）
       → 输入验证码校验
       → 校验通过即为注册成功，自动登录（等级：normal）
       → 后续可在设置中补充密码、绑定其他登录方式、开启 2FA
```

### 1.4 Github OAuth 注册

```
点击「Github 登录」→ 跳转 Github 授权页
                   → 用户点击授权
                   → 回调校验：
                     ├── Github 主邮箱未注册 → 自动创建 normal 账号，登录成功
                     ├── Github 主邮箱已注册 → 自动绑定 Github 方式到已有账号，登录成功
                     └── 用户取消授权 → 提示"授权已取消"，返回登录页
       → 注册/绑定完成（normal 等级）
       → 引导设置 2FA（可跳过）
```

### 1.5 Passkey 通行密钥绑定

```
已登录状态 → 设置页 → 安全 → 点击「创建通行密钥」
                          → 浏览器调用 WebAuthn.create()
                          → 用户通过系统指纹/面容/PIN 验证
                          → 公钥存储于服务端，私钥存储于用户设备
                          → 绑定成功

仅 normal 和 admin 支持，local 不支持
```

## 二、登录流程

### 2.1 账号密码登录

```
登录页 → 输入 用户名/邮箱/手机号 + 密码
       → 校验账号密码：
         ├── 失败 → 提示"账号或密码错误"，显示剩余尝试次数
         │          → 连续错误 5 次 → 账号锁定 15 分钟
         │          → 发送安全提醒（normal/admin 发邮件，local 不发）
         └── 成功 → 判断账户等级：
                    ├── local  → 直接进入应用
                    │            提示：「绑定邮箱/手机即可解锁全部功能」（可关闭）
                    │            点击 → 跳转设置页 → 绑定任一方式 → 自动升级 normal
                    ├── normal → 检查 2FA 状态：
                    │             ├── 未开启 2FA → 直接进入应用
                    │             └── 已开启 2FA → 跳转 2FA 验证页（见 2.6）
                    └── admin  → 强制 2FA：
                                  ├── 已开启 2FA → 跳转 2FA 验证页
                                  └── 未开启 2FA → 强制绑定页（不绑定不能进入）
                                    （此情况发生在其他 admin 提升了无 2FA 的 normal）
```

### 2.2 手机号/邮箱验证码登录

```
登录页 → 输入手机号/邮箱
       → 点击「获取验证码」（60s 冷却，限频：1条/分钟，5条/小时）
       → 输入验证码
       → 校验通过 → 判断账户等级：
                     ├── normal → 检查 2FA（已开启则跳转 2FA 页）
                     └── admin  → 强制 2FA
       → 校验失败 → 提示"验证码错误"，允许重试 3 次
                   → 超限后需重新获取验证码
       （local 账户没有绑定的手机号/邮箱，不支持此方式）
```

### 2.3 Github OAuth 登录

```
点击「Github 登录」→ 跳转 Github 授权页
                   → 用户授权：
                     ├── 回调成功，账号已存在 → 判断等级：
                     │     ├── normal → 检查 2FA
                     │     └── admin  → 强制 2FA
                     ├── 回调成功，账号不存在 → 自动注册 normal + 登录
                     ├── 用户取消授权 → 提示"授权已取消"，返回登录页
                     └── Github API 不可用 → 提示"第三方登录暂不可用，请使用其他方式"
```

### 2.4 Magic Link 无密码登录

```
登录页 → 输入邮箱
       → 点击「发送魔法链接」
       → 系统发送含一次性 token 的链接到邮箱
       → 用户打开邮箱点击链接 → 校验 token：
         ├── 有效（15 分钟内 + 未使用）→ 判断账户等级：
         │     ├── normal → 检查 2FA
         │     └── admin  → 强制 2FA
         ├── 已过期 → 提示"链接已过期，请重新获取"
         └── 已使用 → 提示"链接已失效，请重新获取"

（local 账户无邮箱，不支持此方式）
```

### 2.5 Passkey 通行密钥登录

```
登录页 → 点击「使用通行密钥登录」
       → 浏览器调用 WebAuthn.get()
       → 用户通过系统指纹/面容/PIN 验证
       → 服务端公钥签名校验：
         ├── 通过 → 判断账户等级：
         │     ├── normal → 检查 2FA
         │     └── admin  → 强制 2FA
         └── 失败 → 提示"通行密钥验证失败，请使用其他方式登录"

（local 账户不支持 Passkey）
```

### 2.6 2FA 二次验证

```
主校验通过后，根据账户等级进入不同分支：

normal（已开启 2FA）：
  → 跳转 2FA 验证页
  → 输入 TOTP 6 位动态验证码
  → 校验：
    ├── 通过 → 进入应用
    ├── 失败 → 提示"验证码错误"，剩余重试 2 次
    └── 点击"使用备用恢复码" → 进入恢复码验证
  → 可勾选「信任此设备 30 天」→ 本设备 30 天内免二次验证

admin（强制 2FA）：
  → 如果已开启 2FA → 跳转 2FA 验证页（流程同 normal）
  → 如果未开启 2FA → 强制进入 2FA 绑定流程：
                     ├── 展示 TOTP 密钥/二维码
                     ├── 用户扫码/手动输入绑定
                     ├── 验证绑定（输入 6 位码确认）
                     ├── 下载/复制备用恢复码 → 用户勾选"我已安全保存"
                     └── 绑定完成 → 进入应用
  → **不允许**「信任设备 30 天」（每次登录都必须 2FA）
  → TOTP 时间偏差容忍 ±1 窗口，校验通过后自动修正

local：
  → 无 2FA，直接进入应用
```

## 三、密码找回 / 账号恢复流程

### 按账户等级区分

| 账户等级 | 找回方式   | 说明                                      |
| -------- | ---------- | ----------------------------------------- |
| local    | 不支持     | 无邮箱/手机，忘记密码即账号不可恢复       |
| normal   | 邮箱/手机  | 通过验证码或 Magic Link 重置              |
| admin    | 邮箱 + 2FA | 必须同时通过邮箱/手机验证 **和** 2FA 验证 |

### 3.1 手机号验证码找回（normal/admin）

```
找回页 → 输入绑定的手机号
       → 发送短信验证码
       → 输入验证码校验
       → admin 额外要求 → 输入 TOTP 6 位码
       → 校验全部通过 → 设置新密码（需输入两次）
       → 重置成功 → 所有已登录会话自动失效，需重新登录
```

### 3.2 邮箱验证码找回（normal/admin）

```
找回页 → 输入绑定的邮箱
       → 发送验证码到邮箱
       → 输入验证码校验
       → admin 额外要求 → 输入 TOTP 6 位码
       → 校验全部通过 → 设置新密码 → 重置成功 → 全设备强制下线
```

### 3.3 邮箱 Magic Link 找回（仅 normal）

```
找回页 → 输入邮箱
       → 点击「发送重置链接」
       → 邮箱收到含一次性 token 的重置链接
       → 点击链接 → 直接进入设置新密码页
       → 设置新密码 → 重置成功 → 全设备强制下线

（admin 不支持 Magic Link 找回，必须走 3.2 + 2FA）
```

### 3.4 2FA 丢失恢复

```
normal 用户 2FA 丢失：
  登录页主验证通过 → 2FA 验证页
                    → 点击「无法验证 / 丢失设备」
                    → 输入备用恢复码
                      ├── 通过 → 临时登录 → 引导重新设置 2FA
                      └── 失败 → 通过绑定的邮箱/手机验证
                                 └── 校验通过 → 关闭当前 2FA → 引导重新设置

admin 用户 2FA 丢失：
  登录页主验证通过 → admin 无 2FA，无法登录
                    → 点击「管理员恢复流程」
                    → 通过邮箱/手机验证
                    → 输入备用恢复码
                      ├── 通过 → 临时登录 → 强制重新绑定 2FA（不可跳过）
                      └── 失败 → 联系其他 admin 重置 → 其他 admin 操作后
                                 → 临时登录 → 强制重新绑定 2FA
```

### 3.5 Passkey 设备丢失恢复

```
登录页 → 点击「通行密钥不可用 / 丢失设备」
       → 使用其他已绑定的登录方式登录
       → 登录后进入设置页 → 安全 → 删除旧 Passkey → 重新创建

（仅 normal/admin 支持，local 不支持 Passkey）
```

## 四、异常与边界场景

| 场景                      | local          | normal                        | admin                                          |
| ------------------------- | -------------- | ----------------------------- | ---------------------------------------------- |
| 密码错误超过 5 次         | 锁账号 15 分钟 | 锁账号 15 分钟 + 邮箱安全提醒 | 锁账号 15 分钟 + 邮箱安全提醒 + 通知其他 admin |
| 验证码获取过于频繁        | 无此场景       | 1条/分钟，5条/小时            | 1条/分钟，10条/小时                            |
| 验证码输入错误超过 3 次   | 无此场景       | 当前码作废，重新获取          | 当前码作废，重新获取                           |
| Magic Link 过期/已用      | 无此场景       | 提示过期，打回登录页          | 无此场景（admin 不支持 Magic Link）            |
| Github OAuth 取消         | 无此场景       | 提示取消，返回登录页          | 提示取消，返回登录页                           |
| Passkey 不支持            | 无此场景       | 隐藏入口                      | 隐藏入口                                       |
| 2FA TOTP 时间偏差         | 无             | ±1 窗口（±30s），自动修正     | ±1 窗口（±30s），自动修正                      |
| 重置密码后                | 本地不支持     | 全设备强制下线                | 全设备强制下线                                 |
| 提升为 admin 但未绑定 2FA | 不适用         | 不适用                        | 强制绑定，拒绝登录                             |
| admin 关闭 2FA            | 不适用         | 不适用                        | 自动降级为 normal                              |

## 七、登录流程图

```
登录页
├── 密码登录
│       ├── 输入账号 + 密码
│       ├── 校验 → 通过后判断等级：
│       │     ├── local  → 直接进入 + 提示可升级 normal
│       │     ├── normal → 检查 2FA
│       │     │     ├── 未开启 → 直接进入
│       │     │     └── 已开启 → 2FA 验证页
│       │     └── admin  → 强制 2FA
│       │           ├── 已开启 → 2FA 验证页
│       │           └── 未开启 → 强制绑定 2FA
│       └── 失败 → 5 次锁定
│
├── 验证码登录（仅 normal/admin）
│       ├── 输入手机号/邮箱
│       ├── 获取验证码 → 校验
│       └── 通过 → 判断等级（同密码登录）
│
├── Github 登录（仅 normal/admin）
│       └── OAuth 授权 → 自动注册/绑定 → 判断等级
│
├── Magic Link 登录（仅 normal）
│       ├── 输入邮箱
│       ├── 点击邮件链接
│       └── token 校验 → 检查 2FA → 登录
│
└── Passkey 登录（仅 normal/admin）
        └── 指纹/面容验证 → 判断等级 → 登录

                    ↓ (2FA 接入点，仅 normal 和 admin)

2FA 页面
├── normal：
│     ├── 输入 TOTP 6 位码 → 进入应用
│     ├── 输入备用恢复码 → 进入应用
│     └── 可勾选「信任本设备 30 天」
│
└── admin：
      ├── 输入 TOTP 6 位码 → 进入应用
      ├── 输入备用恢复码 → 进入应用
      └── 不可信任设备（每次必验）
```

## 八、密码找回流程图

```
找回页
├── local
│     └── 不支持找回，忘记密码即账号不可恢复
│
├── normal
│     ├── 手机号找回 → 验证码 → 新密码 → 下线
│     ├── 邮箱找回（验证码）→ 验证码 → 新密码 → 下线
│     ├── 邮箱找回（Magic Link）→ 点击链接 → 新密码 → 下线
│     ├── 2FA 丢失 → 备用恢复码 / 邮箱手机验证 → 重设 2FA
│     └── Passkey 丢失 → 其他方式登录 → 删除旧 → 重建
│
└── admin
      ├── 密码重置 → 邮箱/手机验证 + TOTP → 新密码 → 下线
      ├── 2FA 丢失 → 邮箱/手机验证 + 备用恢复码 → 强制重设 2FA
      │            → 都失败 → 联系其他 admin 协助
      └── Passkey 丢失 → 其他方式登录 + TOTP → 删除旧 → 重建
```
